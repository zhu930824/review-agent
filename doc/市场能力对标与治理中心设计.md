# 市场能力对标与治理中心设计

## 对标结论

同类产品的能力正在从“AI 给评论”演进为“提交前质量治理平台”。Review Agent 当前已具备多模型审查、Pre-PR 门禁、模型策略配置和驾驶舱基础，本轮新增治理中心后，平台能力边界扩展为：

- PR 摘要、行级建议和多模型交叉审查，用于减少 Reviewer 基础往返。
- 质量门禁和新代码策略，用于把 BLOCKER、MAJOR、人工确认转成发布准入条件。
- 安全扫描、密钥依赖风险和 SARIF 导出，用于承接 DevSecOps 和审计场景。
- 团队规则记忆、规则包和工作流模板，用于把项目 SOP、架构边界和业务准入规则沉淀为可复用资产。
- CI 状态检查、GitHub/GitLab、通知和工单同步，用于让审查结果进入实际交付链路。

## 产品能力矩阵

| 能力 | 当前状态 | 业务价值 | 下一步 |
| --- | --- | --- | --- |
| PR 变更摘要 | 已启用 | 让 Reviewer 快速理解影响面 | 接入发布说明和审查记录 |
| 行级审查建议 | 已启用 | 把问题定位到文件和行 | 增强可应用修复建议 |
| PR 问答 | 部分具备 | 支持围绕一次变更持续追问 | 增加 Ask API 和上下文检索 |
| 自动修复 | 待建设 | 从发现问题推进到修复闭环 | 先生成修复草案，人工确认后提交 |
| 质量门禁 | 已启用 | 阻断高风险变更进入主干 | 写回 CI 状态检查 |
| 安全扫描 | 部分具备 | 防止新增漏洞和敏感风险 | 增加安全规则包和 SCA 结果导入 |
| SARIF 导出 | 待建设 | 对接 GitHub code scanning 等生态 | 将 finding 转换为 SARIF result |
| 团队规则记忆 | 部分具备 | 让模型遵循组织实践 | 从 doc/SOP/人工确认沉淀规则 |

## 治理中心模块

新增前端页面：`/governance`

核心区域：

- 市场能力覆盖：展示已启用、部分具备、待建设能力和覆盖率。
- 优先落地项：按业务影响、建设状态和就绪度排序。
- 发布门禁策略包：把安全发布、AI 生成代码保障、审计证据留存编译成能力、策略和人工检查点。
- 集成路线图：分为已可用、下一阶段、后续扩展。
- 工作流模板：日常 Pre-PR、中风险交叉审查、发布质量门禁、架构委员会评审。

## 架构落地

前端新增：

- `types/governance.ts`
- `utils/governanceCatalog.ts`
- `pages/governance/index.vue`
- `tests/governanceCatalog.test.ts`

后端新增迁移：

- `V3__market_governance_capabilities.sql`

该迁移预留四类治理数据：

- `governance_capability`
- `integration_connector`
- `governance_rule_pack`
- `workflow_template`

后续真实集成时，建议优先落地：

1. GitHub/GitLab webhook 接收器。
2. Pre-PR gate 状态回写。
3. SARIF 导出与上传。
4. 安全规则包和依赖扫描结果导入。
5. 自动修复草案与人工确认工作流。
